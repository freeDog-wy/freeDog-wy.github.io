C语言中那些奇奇怪怪的宏

发表于 2024-04-14 分类于 编程语言 ， C 阅读次数：
本文字数： 4.6k 阅读时长 ≈ 8 分钟

今天整理的一下C语言一些奇奇怪怪的宏，给大家瞧一瞧。

TO_STR 添加双引号

通过在一个标识符前添加 # 号，可以给这个标识符加上双引号。

TO_CHAR 添加单引号

a并不是变量，啥也不是，只是通过预处理器给它套了一个单引号，它就变成字符了。

PRIMITIVE_CAT 字符拼接

符号 ## 可以拼接两个标识符。
至于为什么这里给它取名叫PRIMITIVE_CAT宏，是因为宏扩展的一个机制，往下看就知道了。

CAT 字符拼接

可能你会感到奇怪，这不就是套了一层皮吗，有啥区别呢？

但是如果把上面的 CAT 替换成 PRIMITIVE_CAT，看看会发生啥？

报错了
预处理器直接把 X 拿来拼接了，并没有把 X 替换成 a.
那原因是什么呢？

首先宏函数展开有两种情况：

  1. 参数参与了 ## 或者 # 符号的操作
    此时，由于 PRIMITIVE_CAT 宏函数中的参数X Y参与了## 拼接，那么就不会对宏 X Y 进行展开后传入，而是直接传入。 所以拼接结果是 PRIMITIVE_CAT(X,Y) -> XY -> 2.
    拼接后得到了宏XY，直接展开成 2.
  2. 参数没有参与 ## 或者 # 符号的操作
    此时，CAT 宏函数的参数并没有参与拼接，则先对参数 X Y进行展开，分别展开为 0 1，然后传入宏函数。 所以拼接结果是CAT(X,Y) -> PRIMITIVE_CAT(0,1) -> 01 可以看出这两个拼接宏函数，都是很常用的。

什么？你要问这个宏函数原理是啥？

FILED_SIZE 得到一个结构体中字段的大小（字节数）

在我的电脑上int类型4个字节.

  • ptr：结构体字段的指针
